# Agent Instructions

When working in this repository:

- All skills follow the structure defined in `CONTEXT.md`
- SKILL.md files must stay under 100 lines
- Descriptions must include "Use when..." triggers
- Reference files (REFERENCE.md, EXAMPLES.md) are for deep detail only — link from SKILL.md, don't inline
- Never invent VoC data — only use verbatim language from actual customer sources
- When writing copy, always confirm the awareness stage and sophistication level first
- Healthcare copy must pass the compliance checklist in `skills/domain/healthcare-copy/SKILL.md` before output
